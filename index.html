GTK Recent Files Scrubber

Keep your recent files; Hide your recent vices.

Download this project as a .zip file Download this project as a tar.gz file

Having a "Recently Used" list in the GTK+ Open File dialog is a smart and very useful idea. The problem is that, very quickly, it ends up looking like this

[Screenshot TODO: raunchy_porn.jpg, Q3 Fiscal Projections.ods, disgusting_midget_porn.avi, ...]

...or like this...

[Screenshot TODO: bad_twilight_fanfiction.txt, Master Thesis Draft.odt, bad_pony_fanfiction.txt, Jersey Shore episode X.avi]

...or even like this...

[Screenshot TODO: School Play 1999.avi, baby_pictures_03.jpg]

Oh, the horror.

So, you keep running for the privacy panel to wipe your recent files list (if you can even find it) and, sooner or later, you decide that it'd be easier to just turn off the recent files list and that you wouldn't really be losing anything in the process.

GTK+ Recent Files Scrubber takes a slightly different approach., inspired by the HistoryBlock extension for Firefox. With this scrubber, you give it a list of prefixes (folders, essentially) that you don't want to show up in your recent files list, then you get on with your life.

The scrubber will sit in the background, waiting for new files to appear in the list. If they match any of the entries in your blacklist, it kicks them back out. No fuss, no muss. You just set it and forget it.

So what are you waiting for? Install it today and have a Recently Used list that's actually useful.

Features

  • Use it however you want
    • Can run as a lightweight daemon, watching for new MRU list entries and filtering them as they appear.
    • Can run --once and then exit for use in scripts or for testing/debugging your blacklist.
    • Can be used as a scriptable command for purging the MRU list entirely via the --purge option.
    • PLANNED: A GUI mode for removing specific entries without manually editing recently-used.xbel.
  • Cautious
    • Checks to make sure your recently-used.xbel is only readable and writable by you.
    • Your blacklist is hashed so nosy roommates/girlfriends/boyfriends and curious family members will find it much easier to just snoop around the old-fashioned way.
  • Stealthy
    • No GUI, makes no sounds, and appears in process managers as just another innocuous Python script.
    • It's just one file. Install it wherever you want and rename it to whatever you want to make it more stealthy. (You can also specify non-default names for the blacklist file)
    • The contents of the blacklist file are laid out so as to be easily mistaken for something else. (eg. the data store for a "most frequently used" list based on hashes of file contents)
  • Good for any desktop where apps use GTK+ Open dialogs. (GNOME, Xfce, LXDE, anything with GIMP or Inkscape or Pidgin or Firefox or gVim, etc.)

Requirements

  • Python 2.5+
  • GTK 2.10+ (for gtk.RecentManager)
  • PyGTK

Note: Currently only tested on Lubuntu Linux 11.10 and Ubuntu Linux 12.04 but should theoretically work anywhere the requirements are met. (Though the default location for storing the blacklist isn't ideal on Windows)
